<p>Abbiamo già <a href="https://linuxhub.it/articles/howto-installare-lamp-su-ubuntu,-debian-e-derivate">impostato lo stack LAMP su Debian/Ubuntu</a>, ora vogliamo aprire un sito web per la nostra società o semplicemente creare il nostro portfolio personale. Per questa impresa scegliamo <strong>WordPress</strong>, un CMS gratuito ed open source utilizzabile da tutti, anche dai meno esperti vista la sua semplicità.</p><p>In questa guida vediamo appunto come Installare Wordpress via server Apache.</p><h2>Prerequisiti</h2><p>Per poter installare correttamente WordPress sulla distro da voi utilizzata è necessario avere lo <strong>stack LAMP</strong>. Per sapere come aggiungerlo su Debian, Ubuntu e derivate, potete consultare <a href="https://linuxhub.it/articles/howto-installare-lamp-su-ubuntu-debian-e-derivate">questa guida</a>.</p><p>Se vogliamo essere più precisi, i prerequisiti consigliati sono i seguenti:</p><ul>	<li>Apache 2.4</li>	<li>MySQL 5.6 o MariaDB 10.0</li>	<li>PHP 7.3</li></ul><h2>Download</h2><p>Per prima cosa, entrate nella cartella dove desiderate ospitare il vostro sito web realizzato con WordPress. (In questa guida si lavorerà in <em>/var/www/wordpress/</em>)</p><pre><code>cd /var/www/</code></pre><p>Fatto ciò, scarichiamo l'ultima versione di WordPress in italiano utilizzando <em>wget</em>:</p><pre><code>wget https://it.wordpress.org/latest-it_IT.tar.gz</code></pre><p>Scompattiamo il pacchetto compresso:</p><pre><code>tar -xzvf latest-it_IT.tar.gz</code></pre><p>Entriamo nella cartella di WordPress:</p><pre><code>cd /var/www/wordpress/</code></pre><p>Dando il comando <em>ls </em>noteremo molti file assieme ad alcune cartelle, ma procediamo con ordine.</p><h2>Configurazione di Apache2</h2><p>Per poter accedere in futuro al nostro sito con WordPress, abbiamo bisogno di configurare a dovere il web server che utilizziamo, in questo caso<strong> Apache2</strong>, creando un file di configurazione dedicato. Vediamo come fare.</p><p>Creiamo un nuovo file nella cartella <em>/sites-available/</em> di Apache:</p><pre><code>sudo nano /etc/apache2/sites-available/wordpress.conf</code></pre><p>Andiamo a scrivere quanto dovuto nel file:</p><pre><code>&lt;VirtualHost *:80&gt;        ServerName dominio.it        ServerAdmin email@amministratore.com        DocumentRoot /var/www/wordpress        &lt;Directory /var/www/wordpress&gt;            Options FollowSymLinks            AllowOverride Limit Options FileInfo            DirectoryIndex index.php            Require all granted        &lt;/Directory&gt;        ErrorLog ${APACHE_LOG_DIR}/error.log        CustomLog ${APACHE_LOG_DIR}/access.log combined&lt;/VirtualHost&gt;</code></pre><p>Usciamo dall'editor di testo e creiamo un link simbolico in <em>/sites-enabled/</em>:</p><pre><code># Per Debian e Ubuntusudo ln -s /etc/apache2/sites-available/wordpress.conf /etc/apache2/sites-enabled/</code></pre><p>Abilitiamo il nostro sito:</p><pre><code>sudo a2ensite wordpress.conf</code></pre><p>Disabilitiamo la pagina di default di Apache (se non avete toccato nulla):</p><pre><code>sudo a2endissiste 000-default.conf</code></pre><p>Riavviamo Apache:</p><pre><code>sudo systemctl reload apache2</code></pre><p>Apache2 è ora configurato correttamente per poter eseguire WordPress.</p><h2>Configurazione del database</h2><p>WordPress non ha solamente bisogno di una cartella sull'hard disk del vostro server per poter funzionare a dovere, ma anche bisogno di scrivere dei dati in un <strong>database</strong>. Vediamo come andare a creare un utente dedicato esclusivamente al database su cui andrà a lavorare il CMS.</p><h3>Creazione di un utente</h3><p>Eseguiamo l'accesso in MySQL o MariaDB con l'utente superuser (o root, anche se sconsigliato):</p><pre><code>sudo mysql -u nomeutentesuperuser -p</code></pre><p>Creiamo un nuovo utente:</p><pre><code>CREATE USER nomeutente@localhost IDENTIFIED BY 'vostrapassword';</code></pre><h3>Creazione di un database</h3><p>Dopo aver creato l'utente, andiamo a realizzare il database:</p><pre><code>CREATE DATABASE nomedatabasewordpress;</code></pre><p>Diamo tutti i permessi all'utente creato in precedenza per lavorare senza problemi sul database:</p><pre><code>GRANT ALL PRIVILEGES ON nomedatabasewordpress.* TO nomeutente@localhost;</code></pre><p>Informiamo MySQL o MariaDB dei cambiamenti e usciamo:</p><pre><code>FLUSH PRIVILEGES;EXIT;</code></pre><p>Congratulazioni! Avete impostato correttamente un utente e un database per WordPress.</p><h2>Configurazione di wp-config.php</h2><p>Manca poco all'installazione vera e propria di WordPress, ma prima abbiamo ancora bisogno di scrivere nel file di configurazione, chiamato <strong><em>wp-config.php</em></strong>.</p><p>Per prima cosa, copiamo il file d'esempio:</p><pre><code>cp wp-config-sample.php wp-config.php</code></pre><p>Ora, andiamo a sistemare un momento il file di configurazione appena generato. Appena entreremo con il nostro editor di testo noteremo subito dei commenti e dei parametri da modificare, ma procediamo con ordine.</p><p>Cambiamo il nome del database di WordPress in quello appena creato:</p><pre><code>define('DB_NAME', 'nomedatabasewordpress');</code></pre><p>Forniamo a WordPress il nome utente e la password dell'utente del database generati in precedenza.</p><pre><code>define('DB_USER', 'nomeutente');define('DB_PASSWORD', 'password');</code></pre><p>Ora, qui non dobbiamo più modificare nulla, spostiamoci più sotto nella sezione "Chiavi univoche di autenticazione e di salatura". Per una questione di sicurezza, è altamente raccomandato generare delle stringhe da inserire in "Mettere la vostra frase unica qui" nel valore appropriato.</p><p>Generiamo le chiavi da <a href="https://api.wordpress.org/secret-key/1.1/salt/">questo link</a> e copiamo ed incolliamo quanto fornito da WordPress andando a sostituire i parametri già presenti. Pertanto:</p><pre><code>define('AUTH_KEY',         'Mettere la vostra frase unica qui');define('SECURE_AUTH_KEY',  'Mettere la vostra frase unica qui');define('LOGGED_IN_KEY',    'Mettere la vostra frase unica qui');define('NONCE_KEY',        'Mettere la vostra frase unica qui');define('AUTH_SALT',        'Mettere la vostra frase unica qui');define('SECURE_AUTH_SALT', 'Mettere la vostra frase unica qui');define('LOGGED_IN_SALT',   'Mettere la vostra frase unica qui');define('NONCE_SALT',       'Mettere la vostra frase unica qui');</code></pre><p>Si trasformerà in:</p><pre><code>define('AUTH_KEY',         'Valore generato dal link');define('SECURE_AUTH_KEY',  'Valore generato dal link');define('LOGGED_IN_KEY',    'Valore generato dal link');define('NONCE_KEY',        'Valore generato dal link');define('AUTH_SALT',        'Valore generato dal link');define('SECURE_AUTH_SALT', 'Valore generato dal link');define('LOGGED_IN_SALT',   'Valore generato dal link');define('NONCE_SALT',       'Valore generato dal link');</code></pre><p>Ora, usciamo dall'editor di testo e accediamo al file d'installazione di WordPress dal nostro dominio:</p><pre><code>http://dominio.it/wp-admin/install.php</code></pre><h2>Configurazione di WordPress</h2><p>Appena caricata la pagina di installazione di WordPress ci troveremo una cosa simile di fronte:</p><p><img class=" size-full wp-image-552" alt="Pagina di installazione di WordPress" data-entity-type="file" data-entity-uuid="7a6a5866-fd62-4d53-a8e2-267589e0a7e9" src="https://linuxhub.it/wordpress/wp-content/uploads/2019/11/wordpress-installazione.png" width="835" height="883" /></p><p>Inseriamo quanto richiesto (evitando di utilizzare nomi utenti come admin, amministratore, ecc e password deboli), clicchiamo il bottone Installa WordPress e... congratulazioni! <strong>Avete installato WordPress correttamente</strong>. Facciamo l'accesso con i nostri dati ed ecco a voi la dashboard.</p><h2>Conclusione</h2><p>Ora sta a voi imparare ad usare questo potente strumento, impostando tutto più come vi pare e piace. Per fornire più sicurezza al vostro sito web, potete consultare <a href="https://forum.felinesec.com/d/32-mettere-in-sicurezza-un-sito-web-realizzato-con-wordpress">questa guida</a> realizzata da me sul forum di FelineSec, gruppo facente parte del network GenteDiLinux.</p><p>Per dubbi e chiarimenti, utilizzate il nostro&nbsp;<a href="https://t.me/gentedilinux">gruppo Telegram</a>.</p>
