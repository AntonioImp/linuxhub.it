<p><strong>Bludit</strong> è un CMS&nbsp;gratuito e Open source,&nbsp;basato sull'archiviazione in file <strong>JSON</strong>, utile nella realizzazione di blog e siti web veloci e funzionali in breve tempo.</p><p>La sua assenza di un database vero e proprio, come ad esempio MySQL o SQLite, lo rende pronto all'uso in pochi minuti oltre che facilmente migrabile e poco esoso di risorse.</p><h2>Requisiti</h2><p>Per la messa online di Bludit, abbiamo le seguenti necessità:</p><ul>	<li>Un Web server (Nginx o Apache)</li>	<li>Php (5.6 o superiore)</li></ul><p>fra cui ovviamente non è necessario un database SQL.</p><h2>Configurazione dominio</h2><p>Il dominio è sostanzialmente&nbsp;"la base del link" da cui vogliamo raggiungere la nostra installazione. Un esempio è proprio questo sito web, raggiungibile appunto da: linuxhub.it.</p><p>Per una miglior&nbsp;fruibilità dei contenuti, la guida su come <strong>puntare un dominio ad un IP</strong>&nbsp;è disponibile <a href="https://linuxhub.it/article/howto-puntare-un-dominio-ad-un-ip">a questo link</a>.</p><h2>Installazione</h2><p>Ci sono più modi in cui installare questo CMS:</p><ul>	<li>il metodo&nbsp;<strong>classico</strong>&nbsp;in cui andiamo a configurare un web server</li>	<li>il metodo&nbsp;<strong>semplificato</strong>&nbsp;ossia un web server minimale pronto all'uso, messo a disposizione dal team di sviluppo stesso</li></ul><p>non è chiaro se nel secondo caso ci sono dei cali di prestazioni o delle limitazioni ma è importante tenere in considerazione che si tratta più di un server di sviluppo che di produzione.</p><h3>Installazione semplificata</h3><p>Come dal titolo di questa sezione, l'installazione è estremamente semplice e bastano solo pochi minuti per la messa in opera. Per prima cosa installiamo php:</p><pre><code>sudo rpm -Uvh https://mirror.webtatic.com/yum/el7/webtatic-release.rpmsudo yum install -y php72w php72w-cli php72w-fpm php72w-common php72w-mbstring php72w-zip php72w-pgsql php72w-sqlite3 php72w-curl php72w-gd php72w-mysql php72w-intl php72w-json php72w-opcache php72w-xml</code></pre><p>successivamente otteniamo una copia del sorgente:</p><pre><code>git clone https://github.com/bludit/bludit.git</code></pre><p>ed avviamo il server:</p><pre><code>cd bluditphp -S localhost:8000</code></pre><p>dove:</p><ul>	<li><strong>localhost</strong>&nbsp;è il dominio associato al server</li>	<li><strong>8000</strong>&nbsp;la porta (aperta) in cui rendere accessibile il CMS</li></ul><p>Continua nella sezione <strong>Configurazione del CMS</strong>.</p><h3>Installazione classica (Nginx o Apache)</h3><p>Nell'instalazione classica (raccomandata in caso di produzione), procediamo con l'installazione di Nginx o Apache e php-fpm.</p><h4>Installazione di php-fpm</h4><p>Iniziamo dall'installazione di php ed i moduli necessari come da requisiti:</p><pre><code>sudo yum install php php-fpm</code></pre><p>proseguiamo poi con l'installazione dei&nbsp;moduli:</p><pre><code>sudo yum install php-gd php-ldap php-odbc php-pear php-xml php-xmlrpc php-mbstring php-snmp php-soap curl curl-devel php-json</code></pre><p>in questo modo la nostra configurazione sarà compatibile con una installazione base del CMS.</p><p>Infine avviamo il processo ed abilitiamo l'auto esecuzione all'avvio del server:</p><pre><code>sudo systemctl start php-fpmsudo systemctl enable php-fpm</code></pre><h4>Installazione del&nbsp; Web server</h4><p>Il resto di questa sezione&nbsp;si suddivide in due parti, ossia la configurazione di un server Nginx e quella di un server Apache. Importante tenere in considerazione che questi due software possono coesistere ma in questa guida non tratteremo l'installazione di Bludit&nbsp;su una installazione di questo tipo.</p><p>Di seguito sono riportate le guide su come installare i due Web server (Nginx e Apache). Una volta scelto, seguite l'installazione con la guida&nbsp;appropriata, infine procedete con la sezione:&nbsp;<strong>Configurazione Web server.</strong></p><h4>Installazione Nginx</h4><p>Fra le scelte questa è per me la migliore. Se lo scopo del nostro sito web è quello dei grandi numeri in termini di traffico, Nginx è in grado di gestire con facilità grossi carichi, inoltre&nbsp;la sua struttura asincrona gli permette di gestire più eficientemente richieste multiple, occupando la RAM per meno tempo rispetto Apache.</p><p>Per una miglior&nbsp;fruibilità dei contenuti, la guida su come installare <strong>Nginx</strong> su Centos 7 è disponibile <a href="https://linuxhub.it/article/howto-installare-nginx-su-centos-7-e-configurazione-ssl">a questo link</a>.</p><h4>Installazione Apache</h4><p>Per una miglior&nbsp;fruibilità dei contenuti, la guida su come installare <strong>Apache</strong>&nbsp;su Centos 7 è disponibile <a href="https://linuxhub.it/article/howto-installare-apache-su-centos-7">a questo link</a>.</p><h2>Configurazione (Nginx o Apache)</h2><p>Procedi nella lettura solo in caso di installazione personalizzata (Nginx e Apache).</p><p>Una volta installato il web server preferito, possiamo procedere con la sua configurazione e quella dello spazio web che andremo a dedicare al CMS.</p><p>Continua di seguito con la sezione inerente il Web server che hai scelto.</p><h3>Configurazione Nginx</h3><p>Per prima cosa creiamo una locazione da dedicare al CMS:</p><pre><code>mkdir -p /usr/share/nginx/il_mio_dominio_ex</code></pre><p>dove&nbsp;<strong>il_mio_dominio_ex</strong>&nbsp;è il nome del dominio configurato precedentemente.&nbsp;</p><blockquote><p>Non è indispensabile che nome della locazione e dominio siano uguali ma lo consiglio per una migliore gestione futura in caso il server dovesse ospitare più siti web.</p></blockquote><p>Impostiamo ora i giusti permessi al nuovo percorso:</p><pre><code>chowh nginx:nginx /usr/share/nginx/il_mio_dominio_exchmod 775 /usr/share/nginx/il_mio_dominio_ex</code></pre><p>In questa fase andiamo a creare un blocco&nbsp;<strong>server</strong>&nbsp;su Nginx per la nostra installazione. Portiamoci alla locazione&nbsp;<strong>/etc/nginx/conf.d/default.conf</strong>&nbsp;e creiamo&nbsp;un nuovo blocco come la seguente:</p><pre><code>server {    listen       80;    server_name  il_mio_dominio.ex www.il_mio_dominio.ex;    root   /usr/share/nginx/il_mio_dominio_ex;    index index.php index.html index.htm;    location / {        try_files $uri $uri/ =404;    }    error_page 404 /404.html;    error_page 500 502 503 504 /50x.html;    location = /50x.html {        root /usr/share/nginx/html;    }    location ~ \.php$ {        try_files $uri =404;        fastcgi_pass unix:/var/run/php-fpm/php-fpm.sock;        fastcgi_index index.php;        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;        include fastcgi_params;    }}</code></pre><p>dove:</p><ul>	<li><strong>il_mio_dominio.ex</strong> - è ovviamente il dominio in nostro possesso che vogliamo dedicare al nuovo sito web</li>	<li><strong>/usr/share/nginx/html/il_mio_dominio_ex</strong> - è il percorso creato precedentemente dove andremo a posizionare i file del CMS</li></ul><p>Procediamo con la modifica di alcune impostazioni di php-fpm necessarie con Nginx. Andiamo ad aprire il file di configurazione nella locazione <strong>/etc/php-fpm.d/www.conf</strong>, cerchiamo la voce&nbsp;<strong>listen</strong>&nbsp;e modifichiamo come di seguito:</p><pre><code>listen = /var/run/php-fpm/php-fpm.sock</code></pre><p>portiamoci poi alle voci&nbsp;<strong>listen.owner</strong>&nbsp;e&nbsp;<strong>listen.group</strong>&nbsp;rendendole:</p><pre><code>listen.owner = nobodylisten.group = nobody</code></pre><p>infine cerchiamo&nbsp;le voci&nbsp;<strong>user</strong>&nbsp;e&nbsp;<strong>group</strong>&nbsp;ed adattiamole come di seguito:</p><pre><code>user = nginxgroup = nginx</code></pre><p>Non ci resta che riavviare i processi:</p><pre><code>sudo systemctl restart php-fpm nginx</code></pre><h3>Configurazione Apache</h3><p>Per prima cosa creiamo una locazione dedicata ai&nbsp;file del CMS:</p><pre><code>mkdir -p /var/www/il_mio_dominio_ex</code></pre><p>dove <strong>il_mio_dominio_ex</strong> è il nome del nostro dominio.</p><p>Impostiamo poi i permessi:</p><pre><code>chowh apache:apache /var/www/il_mio_dominio_exchmod 775 /var/www/il_mio_dominio_ex</code></pre><p>Andiamo ora a creare la configurazione Apache per il nostro sito web. Creiamo quindi il file alla locazione&nbsp;<strong>/etc/httpd/vhosts.d/il_mio_dominio.ex.conf </strong>col seguente contenuto:</p><pre><code>&lt;VirtualHost localhost:80&gt;ServerAdmin admin@il_mio_dominio.exDocumentRoot "/var/www/html/il_mio_dominio_ex/"ServerName il_mio_dominio.exServerAlias www.il_mio_dominio.exErrorLog "/var/log/httpd/il_mio_dominio.ex-error_log"CustomLog "/var/log/httpd/il_mio_dominio.ex-access_log" combined&lt;Directory "/var/www/html/il_mio_dominio_ex/"&gt;DirectoryIndex index.html index.phpOptions FollowSymLinksAllowOverride AllRequire all granted&lt;/Directory&gt;&lt;/VirtualHost&gt;</code></pre><p>dove:</p><ul>	<li><strong>il_mio_dominio.ex</strong> - è ovviamente il dominio in nostro possesso che vogliamo dedicare al nuovo sito web</li>	<li><strong>/usr/share/nginx/html/il_mio_dominio_ex</strong> - è il percorso creato precedentemente dove andremo a posizionare i file di installazione del CMS</li></ul><p>Andiamo ora a modificare alcune configurazioni per istruire Apache all'uso di&nbsp;php-fpm per la gestione delle richieste php. Creiamo un nuovo file di configurazione alla locazione&nbsp;<strong>/etc/httpd/conf.d/php.conf</strong>&nbsp;col seguente contenuto:</p><pre><code>&lt;Proxy "unix:/var/run/php-fpm/default.sock|fcgi://php-fpm"&gt;    	ProxySet disablereuse=off&lt;/Proxy&gt;&lt;FilesMatch \.php$&gt;	SetHandler proxy:fcgi://php-fpm&lt;/FilesMatch&gt;## Allow php to handle Multiviews#AddType text/html .php## Add index.php to the list of files that will be served as directory# indexes.#DirectoryIndex index.php</code></pre><p>Portiamoci al file&nbsp;<strong>/etc/php-fpm.d/www.conf</strong>&nbsp;e cerchiamo la voce&nbsp;<strong>listen</strong>&nbsp;modificandola come di seguito:</p><pre><code>listen = /var/run/php-fpm/default.sock</code></pre><p>modifichiamo poi le seguenti voci:</p><pre><code>listen.owner = apachelisten.group = apachelisten.mode = 0660user = apachegroup = apache</code></pre><p>Riavviamo infine i processi con:</p><pre><code>systemctl restart php-fpm httpd</code></pre><h2>Installazione del CMS</h2><p>Andiamo ora effettivamente ad installare il CMS sul web server scelto. Per prima cosa ci rechiamo nella locazione che abbiamo precedentemente creato&nbsp;e dedicato allo scopo (differente in Nginx e Apache), nel nostro esempio:</p><pre><code>cd /usr/share/nginx/il_mio_dominio_ex</code></pre><p>Preleviamo dal sito ufficiale (<a href="https://www.bludit.com/">https://www.bludit.com/</a>) il link al download del CMS e scarichiamone la risorsa:</p><pre><code>wget https://www.bludit.com/releases/bludit-3-9-1.zip</code></pre><p>possiamo ora decomprimere l'archivio con&nbsp;<strong>unzip</strong>:</p><pre><code>unzip bludit*zip</code></pre><p>e spostiamo le risorse nella root del sito:</p><pre><code>mv bludit*/* ./</code></pre><h2>Configurazione del CMS</h2><p>Bludit viene offerto di base con una installazione guidata, basterà visitare il dominio o indirizzo IP:</p><ul>	<li>sulla porta 8000 in caso di installazione semplificata</li>	<li>direttamente il dominio configurato in precedenza sulla porta 80 in caso di Nginx o Apache</li></ul><p>per accedere alla prima configurazione del CMS, basterà poi seguire le istruzioni su schermo.</p><p>Consigliamo di proseguire la lettura della <a href="https://docs.bludit.com/en/getting-started/introduction">documentazione ufficiale</a> per quanto riguarda: temi, plugin e contenuti.</p><p>&nbsp;</p><p><em>Good&nbsp;<strong>*nix</strong>?</em><br /><em>&nbsp;- Mirko</em></p>
